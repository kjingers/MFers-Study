# UI Inspiration: 10–15 Popular App Patterns to Borrow

This app is essentially: **weekly schedule + reference content + lightweight coordination**.
Below are mainstream patterns you can borrow (not Bible-specific).

## 1) Google Calendar / Apple Calendar — week-based navigation
- Clear date anchoring, quick jump via a selector
- Use: your “Week of … (Tuesday)” header + jump list

## 2) Todoist / TickTick — compact lists + detail panels
- List of items, tap to view details without losing context
- Use: questions as list items; expand for details

## 3) Notion — blocks + collapsible sections
- Readable typography and collapsible toggles
- Use: reading + questions as “blocks” inside cards

## 4) Slack / Teams — bottom tabs + predictable navigation
- Few top-level destinations, persistent bar
- Use: Week/Meal tabs

## 5) Headspace / Calm — large, calm typography + minimal chrome
- Prioritize readability and whitespace
- Use: Bible passage modal styling

## 6) Airbnb — card-based browsing + bottom sheets
- Smooth bottom sheets for selection/filter
- Use: Week selector as bottom sheet

## 7) Strava — “activity summary” card + expandable detail
- Summary first, details on demand
- Use: reading summary + “view passage” expansion

## 8) Duolingo — focused single task per screen
- Keeps user on the “one thing that matters now”
- Use: default landing is current week, not a dashboard maze

## 9) Spotify — modal sheets + consistent typography scales
- Use: translation picker + modal for verses

## 10) Amazon — clear hierarchy + sticky controls
- Use: sticky bottom tabs; optional sticky “Next question” control in present mode

## 11) Trello — cards + quick edit affordances
- Use: admin edit mode later (inline edits)

## 12) GitHub Mobile — compact headers + overflow menus
- Use: overflow menu for “Copy link”, “Share week”, “Edit week” (admin)

## 13) Weather apps — “today” focus + forecast browsing
- Use: current week focus; browse past/upcoming quickly

## 14) Banking apps — quick actions + safe modals
- Use: “Update headcount” as a quick modal with +/- stepper

## 15) Kindle — reading-first UI
- Use: Bible passage viewer with reader-like controls (font size optional)

### Design best practices sources
- NN/g on mobile navigation patterns and persistent tab bars.
- Material Design navigation guidance.
- For modals/sheets: use an accessible dialog primitive (Radix / React Aria).
